# AGENTS.md

## Project Name

SubtextAI

## Product Mission

Build a premium AI-native SaaS product that helps users decode what people really mean in messages, emails, and conversations.

SubtextAI is NOT a generic chatbot.

It is a communication intelligence engine.

---

## Core Product Promise

Users paste text.

The system reveals:

- hidden intent
- emotional tone
- power dynamics
- interest signals
- red flags
- best strategic reply

---

## Engineering Standard

Every output must feel production-grade.

Prioritize:

- clarity
- maintainability
- speed
- elegance
- premium UX

Avoid:

- bloated code
- generic layouts
- messy component trees
- hacky quick fixes
- ugly spacing

---

## Design Standard

UI should feel like:

- Apple
- Linear
- Notion
- modern funded AI startup

Use:

- premium typography
- strong spacing hierarchy
- elegant dark mode
- subtle motion
- minimal but powerful visuals

---

## Technical Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Framer Motion
- Lucide Icons

---

## Git Workflow

Never commit directly to main.

Use:

- main = stable release
- dev = integration
- feature/* = development branches

---

## Coding Rules

- use reusable components
- strongly typed props
- server/client separation correctly
- modular architecture
- semantic HTML
- accessible forms
- responsive first
- avoid giant files

---

## Product Mindset

Do not build demo pages.

Build real product experiences.

Every page should answer:

Would a real startup ship this?

---

## Performance Rules

- fast initial load
- lazy where useful
- optimized images
- avoid unnecessary client components

---

## AI Rules

AI must feel central to product.

Never bolt AI as decoration.

---

## Final Rule

If two solutions exist:

Choose the cleaner, more premium, more scalable option.
